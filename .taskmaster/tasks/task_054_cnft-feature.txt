# Task ID: 54
# Title: Update API Routes for Bulk cNFT Swap Support
# Status: pending
# Dependencies: None
# Priority: high
# Description: Update the offers.routes.ts API endpoints to support bulk cNFT swaps and enhanced offer management including multi-asset validation, offer updates, counter-offers, cancellation, and multi-transaction response formats.
# Details:
Implement comprehensive API route updates to support bulk cNFT swap operations and enhanced offer management:

1. **Update POST /api/offers for Multi-Asset Support**:
   - Remove existing single-asset validation logic in offers.routes.ts
   - Add validation middleware to enforce maximum 10 assets per side (offeredAssets and requestedAssets arrays)
   - Support mixed asset types (NFTs, cNFTs, SOL) in request body validation
   - Update request schema to accept arrays: { offeredAssets: Asset[], requestedAssets: Asset[], solAmount?: number }
   - Integrate with updated OfferManager.createOffer() method from Task 43

2. **Add PUT /api/offers/:id Endpoint**:
   - Create new route handler for offer updates
   - Validate maker wallet authorization using signature verification middleware
   - Accept updated SOL amounts and asset arrays in request body
   - Call OfferManager.updateOffer() method from Task 49
   - Return updated offer details with version tracking information
   - Add proper error handling for invalid offer states and unauthorized access

3. **Add POST /api/offers/:id/counter-offer Endpoint**:
   - Implement counter-offer creation route handler
   - Accept counter-offer assets and link to parent offer via parentOfferId
   - Validate counter-offer assets are compatible with original offer context
   - Call OfferManager.submitCounterOffer() method from Task 47
   - Return new offer object with parentOfferId relationship
   - Include proper validation for offer chain depth limits

4. **Add DELETE /api/offers/:id Endpoint**:
   - Create cancellation route handler with maker/admin authorization
   - Validate cancellation permissions (maker wallet or admin role)
   - Call OfferManager.cancelOffer() method from Task 48
   - Update offer status to 'cancelled' and set cancelledAt timestamp
   - Return confirmation response with updated offer status
   - Handle nonce account cleanup if offer was in 'pending' state

5. **Update Response Format for Multi-Transaction Swaps**:
   - Modify existing route responses to return transaction group arrays instead of single transactions
   - Include bundleId field for Jito bundle submissions when multiple transactions are required
   - Add status field for each transaction in the group (pending, submitted, confirmed, failed)
   - Update TransactionBuilder integration to handle bulk swap transaction construction
   - Ensure backward compatibility with single-transaction responses

6. **Update OpenAPI/Swagger Documentation**:
   - Update src/public/openapi.json with new endpoint definitions
   - Add request/response schemas for bulk asset arrays and multi-transaction responses
   - Include examples for cNFT assets with isCompressed: true and proof data
   - Document new error codes and validation messages for bulk operations
   - Add authentication requirements for protected endpoints
