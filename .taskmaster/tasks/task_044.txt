# Details:
## Test Scenario
Happy path test for NFT-to-USDC swap on devnet with 3 wallets:
- **Sender Wallet**: Owns NFT, receives USDC payment (minus fee)
- **Receiver Wallet**: Pays USDC, receives NFT
- **Fee Collector Wallet**: Receives 1% fee in USDC

## Prerequisites & Setup

### 1. Wallet Setup
- Verify 3 test wallets exist and are funded with SOL (for transaction fees)
- Check existing setup in `scripts/fund-devnet-wallets.ps1`
- Small values: Use ~0.1 USDC for swap, 0.001 USDC for fee

### 2. USDC Token Setup
**Challenge**: Wallets funded with SOL need USDC tokens
**Solutions**:
a) Use Solana Devnet USDC faucet (if available)
b) Mint devnet USDC tokens using spl-token CLI:
   - Create USDC token mint (or use existing devnet USDC mint)
   - Mint tokens to receiver wallet (~0.2 USDC for testing)
   - Mint tokens to fee collector for initial setup verification
c) Use Solana devnet faucet to airdrop devnet USDC

**Recommended**: Use SPL Token CLI to mint test USDC tokens on devnet

### 3. NFT Creation
**Challenge**: Create NFT in sender's wallet
**Solutions**:
a) Use Metaplex CLI to create test NFT collection and mint
b) Use @metaplex-foundation/js SDK programmatically:
   ```typescript
   import { Metaplex } from '@metaplex-foundation/js';
   const metaplex = Metaplex.make(connection);
   const nft = await metaplex.nfts().create({ ... });
   ```
c) Leverage existing `scripts/setup-nft-collection.ps1` if available

**Recommended**: Use Metaplex JS SDK within test setup to programmatically create NFT

## Test Implementation Steps

### Phase 1: Test Setup
1. Load/verify 3 devnet wallets (sender, receiver, fee-collector)
2. Ensure all wallets have SOL for transaction fees (~0.1 SOL each)
3. Create/mint test NFT in sender wallet using Metaplex
4. Mint devnet USDC tokens to receiver wallet (0.2 USDC)
5. Verify initial balances

### Phase 2: Create Escrow
1. Sender creates escrow transaction via API
2. Deposits NFT into escrow program
3. Specifies USDC amount (0.1 USDC) and receiver wallet
4. System calculates 1% fee (0.001 USDC)
5. Verify escrow state in on-chain program

### Phase 3: Execute Swap
1. Receiver accepts escrow via API
2. Transfers USDC (0.1 USDC) to escrow
3. System distributes:
   - 99% (0.099 USDC) to sender
   - 1% (0.001 USDC) to fee collector
4. NFT transferred to receiver
5. Escrow marked as completed

### Phase 4: Verification
1. Verify NFT ownership changed to receiver
2. Verify sender received 0.099 USDC
3. Verify fee collector received 0.001 USDC
4. Verify escrow status is COMPLETED
5. Verify all transactions on Solana explorer
