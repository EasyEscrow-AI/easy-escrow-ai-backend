# Task ID: 6
# Title: Implement OfferManager Service for Core Swap Business Logic
# Status: done
# Dependencies: 1, 2, 3, 4, 5
# Priority: high
# Description: Create a comprehensive service that manages the complete lifecycle of swap offers including creation, counter-offers, cancellation, acceptance, and fulfillment with proper transaction management and coordination with other core services.
# Details:
Implement the OfferManager service with the following core components:

1. **Offer Creation and Management**:
   - Create maker offers with support for both direct offers (targeted to specific wallet) and open offers (available to any taker)
   - Generate unique offer IDs and store offer details in database with proper indexing
   - Implement offer validation using AssetValidator service to verify maker owns offered assets
   - Calculate and store platform fees using FeeCalculator service
   - Set offer expiration timestamps and implement automatic cleanup of expired offers
   - Support partial fills for divisible assets and track remaining quantities

2. **Counter-Offer System**:
   - Allow users to create counter-offers against existing offers with modified terms
   - Link counter-offers to parent offers and maintain offer chain history
   - Implement counter-offer validation to ensure reasonable terms and asset ownership
   - Handle counter-offer acceptance and rejection workflows
   - Prevent circular counter-offer chains and implement maximum counter-offer depth

3. **Offer Cancellation and Nonce Management**:
   - Implement offer cancellation with proper database state updates
   - Coordinate with NoncePoolManager to invalidate associated nonce accounts
   - Handle batch cancellation for multiple offers
   - Implement maker-initiated cancellation and automatic expiration-based cancellation
   - Clean up related counter-offers when parent offers are cancelled

4. **Offer Acceptance and Transaction Building**:
   - Validate taker eligibility and asset ownership before acceptance
   - Coordinate with TransactionBuilder service to construct atomic swap transactions
   - Assign nonce accounts from NoncePoolManager for transaction durability
   - Return serialized transaction data ready for taker signature and submission
   - Implement transaction size optimization and fee calculation integration
   - Handle ATA creation requirements for both maker and taker

5. **Post-Transaction Management**:
   - Mark offers as filled after receiving on-chain confirmation
   - Update user swap statistics and transaction history
   - Release nonce accounts back to pool after successful completion
   - Handle failed transactions and implement retry logic for network issues
   - Maintain audit trail of all offer state changes

6. **Database Transaction Management**:
   - Implement proper database transactions for all multi-step operations
   - Use database locks to prevent race conditions during offer acceptance
   - Implement rollback mechanisms for failed operations
   - Ensure data consistency across related tables (offers, users, transactions)
   - Handle concurrent access scenarios with proper isolation levels

7. **Service Coordination**:
   - Integrate with NoncePoolManager for nonce account lifecycle management
   - Use AssetValidator for all asset ownership verification operations
   - Leverage FeeCalculator for accurate fee computation and validation
   - Coordinate with TransactionBuilder for transaction construction
   - Implement proper error handling and service dependency management
