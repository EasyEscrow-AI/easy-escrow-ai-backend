# Task ID: 59
# Title: Implement Proof Freshness Strategy for Bulk cNFT Swaps
# Status: pending
# Dependencies: 42, 58
# Priority: high
# Description: Implement comprehensive proof freshness management for bulk cNFT swaps to prevent stale Merkle proof errors through just-in-time fetching, validation, and automatic refresh mechanisms.
# Details:
Implement comprehensive proof freshness strategy for bulk cNFT swaps with the following components:

1. **Just-in-Time Proof Fetching in CnftService**:
   - Modify getCnftProof() method to accept optional fetchTime parameter for tracking
   - Implement fetchProofJIT() method that fetches proofs immediately before transaction building
   - Add proof caching with 30-second TTL using Map<string, {proof: any, timestamp: number}>
   - Create ProofFreshnessConfig interface with expirySeconds, maxRetries, refreshThreshold

2. **Proof Validation Before Transaction Building**:
   - Add validateProofFreshness() method in CnftService to check root matches on-chain
   - Implement getCurrentTreeRoot() using Bubblegum program account fetching
   - Add proof validation step in TransactionBuilder.buildAssetTransfers() before instruction creation
   - Create ProofValidationResult interface with isValid, currentRoot, proofRoot, staleness flags

3. **Automatic Proof Refresh on Stale Errors**:
   - Implement refreshStaleProof() method with exponential backoff retry logic
   - Add error detection for "InvalidProof" and "TreeRootMismatch" transaction errors
   - Create proof refresh queue with priority handling for failed transactions
   - Add ProofRefreshStrategy enum (IMMEDIATE, BATCHED, SCHEDULED)

4. **Proof Expiry Timestamp Tracking**:
   - Add proofTimestamp and expiresAt fields to proof cache entries
   - Implement isProofStale() utility function checking 30-second expiry window
   - Create proof cleanup job to remove expired entries from cache
   - Add ProofMetadata interface with fetchTime, treeRoot, canopyDepth, expiresAt

5. **Rebuild-Transaction Endpoint**:
   - Create POST /api/offers/:id/rebuild-transaction endpoint in offers.routes.ts
   - Implement rebuildFailedTransaction() method in offerManager.ts
   - Add transaction failure reason detection and appropriate rebuild strategy
   - Support both single transaction and bundle transaction rebuilding

6. **Concurrent Tree Modification Detection**:
   - Implement detectConcurrentModifications() using tree sequence number tracking
   - Add tree state monitoring during bulk swap execution
   - Create TreeStateMonitor class with real-time change detection
   - Implement transaction sequencing to minimize proof staleness window

7. **Transaction Sequencing Optimization**:
   - Modify TransactionGroupBuilder to order transactions by proof freshness priority
   - Implement proof-aware transaction batching with freshness scoring
   - Add sequential execution mode for high-value swaps requiring maximum freshness
   - Create ProofFreshnessScore calculation based on tree activity and proof age

8. **Architecture Documentation**:
   - Document proof freshness strategy in docs/PROOF_FRESHNESS_STRATEGY.md
   - Include proof lifecycle diagrams and freshness decision trees
   - Add troubleshooting guide for common proof staleness scenarios
   - Document configuration options and performance tuning recommendations
