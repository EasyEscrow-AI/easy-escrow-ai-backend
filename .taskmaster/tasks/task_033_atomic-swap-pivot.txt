# Task ID: 33
# Title: Production Deployment: Treasury PDA Setup and Configuration
# Status: pending
# Dependencies: 31, 32
# Priority: high
# Description: Set up and configure the Treasury PDA on Solana mainnet for atomic swap fee collection in production.
# Details:
This task involves creating and configuring the Treasury PDA for the atomic swap platform on Solana mainnet, ensuring it is ready to collect and manage platform fees. Key steps include:

1. **Generate or Use Production Treasury Authority Keypair**:
   - Use an existing production treasury authority keypair or generate a new one using `solana-keygen new --outfile wallets/treasury-authority.json`.
   - Ensure the keypair is securely stored and backed up.

2. **Derive Treasury PDA**:
   - Use the production platform authority (program ID) and a canonical seed (e.g., "treasury") to derive the PDA address.
   - Example in TypeScript:
     ```typescript
     import { PublicKey } from "@solana/web3.js";
     const TREASURY_SEED = Buffer.from("treasury");
     const [treasuryPda] = PublicKey.findProgramAddressSync(
       [TREASURY_SEED],
       platformProgramId
     );
     console.log("Treasury PDA:", treasuryPda.toBase58());
     ```

3. **Initialize Treasury Account**:
   - Call the `initialize_treasury` instruction from the atomic swap program, passing the derived PDA and the treasury authority.
   - Ensure the transaction is signed by the treasury authority keypair.

4. **Verify Treasury PDA Creation**:
   - Query the account on-chain using `solana account <treasury_pda_address>` to confirm it exists and has the correct owner and data.
   - Use Solana Explorer or Solscan to verify the account is visible and initialized.

5. **Fund Treasury PDA**:
   - If required, fund the Treasury PDA with enough SOL to cover rent exemption using `solana transfer <treasury_pda_address> <amount>`.

6. **Test Treasury Fee Reception**:
   - Execute a test swap transaction that should result in a fee being sent to the Treasury PDA.
   - Verify the fee is received by checking the account balance before and after the transaction.

7. **Document Treasury PDA Address and Authority**:
   - Record the Treasury PDA address and authority in the production documentation.
   - Update the deployment runbook with the PDA address and initialization steps.

8. **Update Production Environment Variables**:
   - Add the Treasury PDA address to production environment variables (e.g., `PROD_TREASURY_PDA_ADDRESS`).
   - Ensure backend services and scripts reference the correct PDA address for fee collection.

Best practices:
- Use secure key management for the treasury authority keypair.
- Double-check all addresses and seeds before deployment.
- Test all steps in a staging environment before production.
- Document all steps and addresses for audit and recovery purposes.

# Test Strategy:
1. **PDA Derivation Verification**:
   - Confirm the Treasury PDA address matches the one derived from the canonical seed and program ID.

2. **Account Initialization Verification**:
   - Query the Treasury PDA account on-chain and confirm it is initialized and owned by the correct program.

3. **Funding Verification**:
   - Check the Treasury PDA account balance after funding to ensure the correct amount was transferred.

4. **Fee Collection Test**:
   - Execute a test swap transaction and verify the fee is sent to the Treasury PDA.
   - Check the account balance before and after the transaction to confirm the fee was received.

5. **Environment Variable Verification**:
   - Confirm the Treasury PDA address is correctly set in production environment variables and referenced by backend services.

6. **Documentation Review**:
   - Verify all steps and addresses are documented in the deployment runbook and production documentation.
