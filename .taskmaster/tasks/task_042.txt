# Task ID: 42
# Title: Setup Production Database Infrastructure on DigitalOcean
# Status: done
# Dependencies: 23
# Priority: high
# Description: Implement simple, safe MVP production database layout using DO managed services: Managed Postgres cluster with three logical databases (prod/stage/dev), Managed Redis for idempotency and queues, DO Spaces for receipts, with proper security, PITR backups, connection pooling, and least-privilege roles
# Details:
**Backend API:** DO App Platform (`api.easyescrow.ai`)

**Database:** DO Managed Postgres in same region with three logical DBs in one cluster:
- `easyescrow_prod` (real users)
- `easyescrow_stage` (staging/smoke on devnet)
- `easyescrow_dev` (playground)

**Cache/Queues:** DO Managed Redis for idempotency keys, webhooks, retries

**Storage:** DO Spaces (S3-compatible) for receipts/artifacts

**Key Requirements:**
- Enable private networking between App Platform and managed services
- Turn on PITR (point-in-time recovery) + daily snapshots
- Create least-privilege roles: `app_user_prod` (R/W, no DDL), `migrate_user_prod` (DDL only), equivalent for stage/dev
- Enable PgBouncer connection pooling
- Configure migration pipeline gated behind `migrate_user_*` roles
- Store secrets via App Platform env vars with quarterly rotation policy

# Test Strategy:
Validate database setup with connection tests from App Platform, verify role permissions, test PITR recovery, validate migration process in stage environment before prod

# Subtasks:
## 1. Create DigitalOcean Managed PostgreSQL Cluster [done]
### Dependencies: None
### Description: Set up a managed PostgreSQL cluster on DigitalOcean with appropriate sizing and configuration for production workloads
### Details:
Create a DO Managed PostgreSQL cluster in the same region as the App Platform. Configure cluster with appropriate node size, enable high availability, and set up in a VPC for private networking. Enable connection pooling with PgBouncer.

## 2. Configure Database Security and Backup Settings [done]
### Dependencies: 42.1
### Description: Enable PITR backups, configure daily snapshots, and set up security settings for the PostgreSQL cluster
### Details:

## 3. Create Logical Databases and User Roles [done]

### Description: Set up three logical databases (prod/stage/dev) and create least-privilege user roles for each environment
### Details:
Create easyescrow_prod, easyescrow_stage, and easyescrow_dev databases. Create role pairs for each: app_user_* (read/write, no DDL) and migrate_user_* (DDL only). Configure proper permissions and access controls for each role.

### Details:
