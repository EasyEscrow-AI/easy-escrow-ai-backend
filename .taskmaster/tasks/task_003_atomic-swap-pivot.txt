# Task ID: 3
# Title: Implement FeeCalculator Service for Platform Fee Logic
# Status: done
# Dependencies: 1
# Priority: medium
# Description: Create a comprehensive fee calculation service that handles different swap types with configurable fee structures, validation against maximum caps, and detailed fee breakdowns.
# Details:
Implement the FeeCalculator service with the following core components:

1. **Fee Structure Configuration**:
   - Load fee configuration from environment variables (FLAT_NFT_FEE, SOL_SWAP_PERCENTAGE, MAX_FEE_CAP_SOL)
   - Support different fee types: flat fees for pure NFT swaps, percentage-based for SOL-involved swaps
   - Implement configurable maximum fee caps (e.g., 0.5 SOL maximum)
   - Create fee rule definitions for different swap scenarios (NFT-to-NFT, NFT-to-SOL, SOL-to-NFT)

2. **Core Fee Calculation Logic**:
   - Implement calculateSwapFee(swapType, assetValues, swapDetails) method
   - For pure NFT swaps: apply flat fee structure based on asset count or complexity
   - For SOL-involved swaps: calculate percentage-based fees on SOL amounts
   - Apply fee caps and validate against maximum limits
   - Handle edge cases like zero-value swaps or minimum fee thresholds

3. **Fee Breakdown and Conversion**:
   - Provide detailed fee breakdown with lamports and SOL conversions
   - Include fee rate information and calculation methodology
   - Return structured fee object with: { totalFeeLamports, totalFeeSOL, feeRate, feeType, breakdown, cappedAmount }
   - Support fee preview functionality for UI display

4. **Validation and Error Handling**:
   - Validate input parameters and swap configurations
   - Ensure fee calculations don't exceed maximum caps
   - Handle invalid swap types or malformed asset data
   - Provide clear error messages for fee calculation failures

5. **Integration Points**:
   - Design service to integrate with swap offer creation and validation
   - Support fee estimation for different user scenarios
   - Provide methods for fee adjustment based on user subsidization status

# Test Strategy:
1. **Unit Testing**:
   - Test fee calculations for all swap types (NFT-to-NFT, NFT-to-SOL, SOL-to-NFT) with various asset values
   - Verify percentage calculations and flat fee applications are accurate
   - Test fee cap enforcement with edge cases (fees exceeding caps, zero fees, minimum thresholds)
   - Validate lamports to SOL conversions and fee breakdown accuracy
   - Test configuration loading from environment variables with different scenarios

2. **Integration Testing**:
   - Test fee calculation integration with mock swap offer data
   - Verify fee service works correctly with different user subsidization statuses
   - Test fee estimation accuracy against actual swap scenarios
   - Validate error handling with malformed input data

3. **Configuration Testing**:
   - Test service behavior with different environment variable configurations
   - Verify fee rules apply correctly for various swap scenarios
   - Test dynamic fee adjustment based on configuration changes

4. **Performance Testing**:
   - Benchmark fee calculation performance with large asset lists
   - Test concurrent fee calculations for multiple swap requests
   - Verify memory usage and calculation efficiency
