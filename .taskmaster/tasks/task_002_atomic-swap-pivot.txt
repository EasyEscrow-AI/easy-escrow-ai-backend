# Task ID: 2
# Title: Implement NoncePoolManager Service for Durable Nonce Account Management
# Status: done
# Dependencies: 1
# Priority: high
# Description: Create a comprehensive service to manage a pool of Solana durable nonce accounts with thread-safe operations for initialization, assignment, retrieval, advancement, and cleanup.
# Details:
Implement the NoncePoolManager service with the following core components:

1. **Pool Initialization and Replenishment**:
   - Create configurable pool size with minimum/maximum thresholds
   - Generate new durable nonce accounts using Solana Web3.js
   - Fund accounts with minimum rent-exempt balance
   - Store account keypairs securely and track pool status
   - Implement automatic replenishment when pool falls below threshold

2. **Thread-Safe Pool Operations**:
   - Use mutex/semaphore locking for concurrent access protection
   - Implement queue-based assignment system to prevent race conditions
   - Add connection pooling for Solana RPC calls
   - Handle pool exhaustion gracefully with waiting mechanisms

3. **User Assignment Logic**:
   - Assign available nonce accounts to users on first request
   - Track subsidy eligibility and mark first assignments appropriately
   - Update user records in database with assigned nonce account
   - Implement assignment timeout and retry mechanisms

4. **Nonce Value Management**:
   - Retrieve current nonce values from on-chain accounts using getAccountInfo
   - Cache nonce values with TTL to reduce RPC calls
   - Handle network failures and stale data scenarios
   - Validate nonce format and account state

5. **Nonce Advancement**:
   - Advance nonce accounts to invalidate pending transactions
   - Use SystemProgram.nonceAdvance instruction
   - Handle advancement failures and retry logic
   - Track advancement history for debugging

6. **Cleanup and Maintenance**:
   - Identify expired/unused nonce accounts based on configurable criteria
   - Safely reclaim accounts by checking for pending transactions
   - Return reclaimed accounts to available pool
   - Implement periodic cleanup job with configurable intervals
   - Log cleanup activities for monitoring

7. **Error Handling and Monitoring**:
   - Comprehensive error handling for RPC failures, network issues
   - Metrics collection for pool utilization, assignment rates
   - Health check endpoints for service monitoring
   - Structured logging for debugging and audit trails

# Test Strategy:

4. **Database Integration Testing**:

5. **Monitoring and Observability**:
   - Verify metrics are collected and exposed correctly
   - Test health check endpoints return accurate pool status
   - Validate log output contains necessary debugging information
   - Test alerting for pool exhaustion scenarios
