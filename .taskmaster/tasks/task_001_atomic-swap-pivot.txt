# Details:
Update existing Prisma infrastructure to support atomic swap system:

**Prisma Integration**:
- Update existing schema.prisma file in /prisma directory
- Create new Prisma migrations using: npm run db:migrate (for dev) or npm run db:migrate:deploy (for staging/production)
- Keep existing tables (they will be commented out later in Task 10)
- Migrations will auto-run on DigitalOcean during deployment via pre-deploy job

**New Tables for Atomic Swaps**:

1. **Users Table**:
   - wallet_address (String, @id, unique Solana wallet address)
   - nonce_account (String?, nullable, references to durable nonce account)
   - is_subsidized (Boolean, @default(false), indicates if user has subsidized transactions)
   - swap_stats (Json, stores aggregated swap statistics like total_swaps, successful_swaps, failed_swaps, total_volume)
   - created_at, updated_at (DateTime, @default(now()), @updatedAt)
   - @@index on wallet_address, nonce_account

2. **Nonce Pool Table**:
   - id (Int, @id, @default(autoincrement()))
   - nonce_account (String, @unique, the durable nonce account public key)
   - status (NonceStatus enum: AVAILABLE, IN_USE, EXPIRED, INVALID)
   - last_used_at (DateTime?, nullable)
   - created_at, updated_at (DateTime, @default(now()), @updatedAt)
   - @@index on status for efficient querying of available nonces

3. **Swap Offers Table**:
   - id (Int, @id, @default(autoincrement()))
   - maker_wallet (String, references Users.wallet_address)
   - offer_type (OfferType enum: MAKER_OFFER, COUNTER_OFFER)
   - parent_offer_id (Int?, nullable, self-referencing for counter-offers)
   - offered_assets (Json array, structure: [{type: 'nft'|'cnft', mint: 'address', metadata: {}}])
   - requested_assets (Json array, same structure as offered_assets)
   - status (OfferStatus enum: ACTIVE, MATCHED, CANCELLED, EXPIRED, COMPLETED)
   - expires_at (DateTime)
   - nonce_account (String, references NoncePool.nonce_account)
   - created_at, updated_at (DateTime, @default(now()), @updatedAt)
   - @@index on maker_wallet, status, expires_at, parent_offer_id

4. **Swap Transactions Table**:
   - id (Int, @id, @default(autoincrement()))
   - offer_id (Int, references SwapOffers.id)
   - counter_offer_id (Int?, nullable, references SwapOffers.id)
   - maker_wallet (String, references Users.wallet_address)
   - taker_wallet (String, references Users.wallet_address)
   - transaction_signature (String, @unique, Solana transaction signature)
   - status (TransactionStatus enum: PENDING, CONFIRMED, FAILED, CANCELLED)
   - gas_fee (BigInt, fee in lamports)
   - is_subsidized (Boolean, whether transaction was subsidized)
   - error_message (String?, nullable, for failed transactions)
   - confirmed_at (DateTime?, nullable)
   - created_at, updated_at (DateTime, @default(now()), @updatedAt)
   - @@index on offer_id, maker_wallet, taker_wallet, status, transaction_signature

**Prisma Migration Considerations**:
- Use proper @relation directives for foreign key relationships
- Define custom validation using @map for database constraints
- Ensure Json fields validate required structure in application layer
- Create appropriate indexes using @@index directives
- Set up proper enum types for status fields
