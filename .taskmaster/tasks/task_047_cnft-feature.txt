# Task ID: 47
# Title: Phase 3b - Enhanced Offer Management: Implement Counter-Offer Functionality
# Status: pending
# Dependencies: None
# Priority: medium
# Description: Implement comprehensive counter-offer functionality allowing users to submit counter-offers for existing swap offers with proper validation, chain tracking, and workflow management.
# Details:
Implement the counter-offer functionality with the following comprehensive components:

1. **Database Schema Verification**:
   - Verify SwapOffer.parentOfferId field exists as Int? in prisma/schema.prisma for offer chain tracking
   - Ensure proper foreign key relationship to parent SwapOffer records
   - Confirm database supports recursive offer chain queries

2. **Counter-Offer Creation Logic**:
   - Implement submitCounterOffer() method in src/services/offerManager.ts
   - Accept parameters: originalOfferId, counterOfferAssets, makerWallet, expirationTime
   - Validate counter-offer assets against original offer context (ensure reasonable asset types)
   - Link counter-offers to parent using parentOfferId field
   - Generate unique counter-offer ID and set appropriate status (PENDING)

3. **Offer Chain Management**:
   - Implement getOfferChain() method to traverse counter-offer chains using parentOfferId
   - Create recursive query logic to fetch complete offer history from root to latest counter-offer
   - Add chain depth validation to prevent excessive counter-offer nesting (max 10 levels)
   - Implement chain status tracking to identify active vs expired offers in chain

4. **Original Offer Status Updates**:
   - Update original offer status to COUNTER_OFFERED when counter-offer is submitted
   - Implement status transition validation (only ACTIVE offers can receive counter-offers)
   - Add timestamp tracking for counter-offer submission events
   - Ensure original offer remains accessible for reference but marked as superseded

5. **API Endpoint Implementation**:
   - Create POST /api/offers/:id/counter-offer endpoint in src/routes/offers.routes.ts
   - Implement request validation for counter-offer parameters
   - Add authentication middleware to verify maker wallet ownership
   - Return complete counter-offer details including chain position and parent reference

6. **Counter-Offer Workflow Management**:
   - Implement acceptCounterOffer() and rejectCounterOffer() methods
   - Handle counter-offer acceptance by creating atomic swap transaction
   - Manage counter-offer rejection with proper status updates and chain cleanup
   - Add expiration handling for counter-offers with automatic status updates

7. **Notification Hooks Preparation**:
   - Add event emission points for counter-offer submission, acceptance, and rejection
   - Implement structured event data format for future notification system integration
   - Create placeholder hooks in offerManager.ts for notification service calls

8. **API Documentation Updates**:
   - Document counter-offer flow in API documentation
   - Include request/response schemas for counter-offer endpoints
   - Add examples showing complete counter-offer chain interactions
   - Document error codes and validation rules for counter-offer operations
