# Task ID: 43
# Title: Remove 1-NFT-Per-Side Restriction in OfferManager for Bulk Swap Support
# Status: pending
# Dependencies: 41
# Priority: high
# Description: Update the OfferManager service to support bulk NFT swaps by removing the 1-NFT-per-side limitation and implementing support for up to 10 assets per side with proper validation and fee calculation.
# Details:
Implement bulk swap support in OfferManager with the following comprehensive updates:

1. **Remove Multi-Asset Restrictions**:
   - Update createOffer() method in src/services/offerManager.ts to remove existing 1-NFT-per-side validation
   - Replace single asset validation with array-based asset validation logic
   - Ensure backward compatibility with existing single-asset offers

2. **Implement New Asset Limits**:
   - Add MAX_ASSETS_PER_SIDE constant set to 10 for both offered and requested assets
   - Implement validation to reject offers exceeding the 10-asset limit per side
   - Add clear error messages for limit violations

3. **Enhanced Asset Validation**:
   - Update asset validation logic to handle arrays of multiple assets (NFTs, cNFTs, SOL)
   - Implement validation for mixed asset type combinations (cNFT + standard NFT + SOL)
   - Ensure each asset in the array passes individual validation (ownership, metadata, proof data for cNFTs)
   - Add duplicate asset detection within the same offer side

4. **Bulk Fee Calculation Integration**:
   - Update fee calculation calls to handle multiple assets by summing individual asset values
   - Integrate with existing FeeCalculator service to process bulk asset arrays
   - Ensure proper fee calculation for mixed asset types (flat fees for NFTs, percentage for SOL)

5. **Database Storage Optimization**:
   - Verify SwapOffer.offeredAssets and SwapOffer.requestedAssets Json fields handle bulk arrays
   - Update offer serialization/deserialization to properly handle multi-asset structures
   - Ensure database queries efficiently handle bulk asset data

6. **Offer Management Updates**:
   - Update offer listing methods to display multi-asset offers with proper asset summaries
   - Modify offer retrieval to handle bulk asset metadata and display information
   - Update offer status tracking to account for multi-asset complexity

7. **Comprehensive Validation Rules**:
   - Implement validation for valid bulk swap combinations (ensure at least one asset per side)
   - Add business logic validation for reasonable swap combinations
   - Ensure total transaction size considerations for bulk swaps
