# Task ID: 32
# Title: Production Deployment: Solana Program Deployment to Mainnet
# Status: pending
# Dependencies: 31, 30
# Priority: high
# Description: Deploy the atomic swap Solana program to mainnet-beta with proper IDL management and production-grade verification.
# Details:
This task involves deploying the atomic swap program to mainnet-beta, ensuring all production best practices are followed. Key steps include:

1. **Build for Mainnet-beta**:
   - Use the appropriate cfg feature flag to build the program for mainnet-beta, ensuring no staging or devnet-specific code is included.
   - Run `cargo build-sbf` from the `programs/escrow/` directory to generate the production-ready `.so` file.

2. **Program Size Verification and Optimization**:
   - Check the compiled program size using `ls -la target/deploy/escrow.so`.
   - If the program exceeds the recommended size limit (typically 500KB for mainnet), optimize by removing unused dependencies, minimizing code, or splitting logic into separate programs.

3. **Deploy to Mainnet**:
   - Set the Solana CLI to mainnet-beta: `solana config set --url mainnet-beta`.
   - Deploy the program using the production deployer wallet: `solana program deploy target/deploy/escrow.so --keypair production-deployer.json`.
   - Record the deployment transaction ID and program ID for documentation.

4. **IDL Management**:
   - Close the old IDL on mainnet using `anchor idl close` if necessary.
   - Generate a new IDL with `anchor idl build` and upload it with `anchor idl upgrade --provider.cluster mainnet-beta --provider.wallet production-deployer.json`.
   - Ensure the buffer size is sufficient for the new IDL to avoid staging issues.

5. **Verify Deployment and IDL Accessibility**:
   - Use Solana Explorer or Solscan to verify the program is deployed and accessible.
   - Confirm the IDL is correctly uploaded and accessible via the Anchor IDL endpoint.

6. **Update Backend IDL Files**:
   - Copy the new IDL to `src/generated/anchor/escrow-idl-production.json`.
   - Commit and push the updated IDL file to the repository.

7. **Test Program Initialization and Basic Functionality**:
   - Initialize the program on mainnet and run basic swap tests to ensure functionality.
   - Verify all swap types (NFT-to-NFT, NFT-to-SOL, SOL-to-NFT, cNFT swaps) work as expected.

8. **Documentation**:
   - Document the program ID and deployment transaction ID in the deployment log.
   - Update any relevant documentation to reflect the production deployment.
