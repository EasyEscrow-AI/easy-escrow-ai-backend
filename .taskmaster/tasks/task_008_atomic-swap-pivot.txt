# Task ID: 8
# Title: Implement Express HTTP API Routes for Atomic Swaps
# Status: done
# Dependencies: 1, 6
# Priority: high
# Description: Create comprehensive Express.js HTTP API endpoints for atomic swap operations including offer creation, counter-offers, cancellation, acceptance, confirmation, and listing with proper validation and error handling.
# Details:
Implement the Express HTTP API routes with the following core components:

1. **Route Structure and Middleware Setup**:
   - Create Express router with proper middleware chain (body parsing, CORS, rate limiting)
   - Implement authentication middleware to verify wallet signatures for protected endpoints
   - Add request logging and error handling middleware with structured error responses
   - Set up input validation middleware using Joi or similar library for request body/params validation

2. **Offer Management Routes**:
   - POST /api/offers: Create new swap offer with maker assets, requested assets, and optional taker specification
     * Validate maker wallet signature and asset ownership
     * Use OfferManager service to create offer and calculate fees
     * Return offer ID and details with 201 status
   - POST /api/offers/:id/counter: Create counter-offer for existing offer
     * Validate taker eligibility and asset ownership
     * Generate counter-offer with modified terms
     * Return counter-offer details with 201 status
   - POST /api/offers/:id/cancel: Cancel existing offer (maker only)
     * Verify maker authorization and offer status
     * Update offer status to cancelled
     * Return confirmation with 200 status

3. **Transaction Execution Routes**:
   - POST /api/offers/:id/accept: Accept offer and return serialized transaction
     * Validate taker authorization and offer availability
     * Use TransactionBuilder service to construct atomic swap transaction
     * Assign nonce account from NoncePoolManager
     * Return serialized transaction for client signing with 200 status
   - POST /api/offers/:id/confirm: Confirm on-chain transaction completion
     * Validate transaction signature and verify on-chain execution
     * Update offer status to completed and release nonce account
     * Return confirmation details with 200 status

4. **Query Routes**:
   - GET /api/offers: List offers with filtering and pagination
     * Support query parameters: maker, taker, status, asset_type, limit, offset
     * Implement proper SQL queries with indexes for performance
     * Return paginated results with metadata
   - GET /api/offers/:id: Get detailed offer information
     * Return complete offer details including assets, fees, and status history
     * Include related counter-offers if applicable

5. **Error Handling and Status Codes**:
   - 400 Bad Request: Invalid input data, malformed requests
   - 401 Unauthorized: Missing or invalid wallet signature
   - 403 Forbidden: Insufficient permissions for operation
   - 404 Not Found: Offer not found or invalid offer ID
   - 409 Conflict: Offer already accepted/cancelled, invalid state transition
   - 422 Unprocessable Entity: Valid format but business logic validation failed
   - 500 Internal Server Error: Service failures, database errors

6. **Input Validation Schemas**:
   - Validate Solana wallet addresses format (base58, 32-44 characters)
   - Validate asset structures (mint addresses, amounts, metadata)
   - Validate offer IDs (UUID format)
   - Sanitize and validate all user inputs to prevent injection attacks
