# Task ID: 43
# Title: Migrate from Upstash to Redis Cloud for Bull Queue Compatibility
# Status: done
# Dependencies: None
# Priority: medium
# Description: Migrate Redis infrastructure from Upstash to Redis Cloud (redis.io) due to Bull job queue incompatibility with Upstash's Lua script limitations. Redis Cloud provides full Redis compatibility required for Bull's dynamic key generation in Lua scripts.
# Details:
## Problem
- Bull job queues are fundamentally incompatible with Upstash Redis
- Upstash requires all Lua script keys as explicit arguments, but Bull generates dynamic keys inside scripts
- Current error rate: 16.5% (EPIPE/ECONNRESET)
- Deployments failing at 4/6 (runtime errors, no logs)
- Jobs processing unreliably

## Solution
Migrate to Redis Cloud Free Tier (redis.io):
- Full Bull/BullMQ compatibility
- 30GB storage, 30 connections (free forever)
- AWS Singapore region (ap-southeast-1) for low latency to DO apps
- IP allowlist + TLS for cross-provider security
- AOF persistence for data reliability

## Expected Results
- 0% error rate (down from 16.5%)
- Successful deployments (6/6 phases)
- Reliable Bull queue processing
- All 3 Bull connections working (client, subscriber, bclient)

## Security Requirements
- TLS encryption (rediss:// protocol)
- IP allowlist configured for DigitalOcean app IPs
- Strong password authentication
- Regular password rotation (monthly)

## Environments
- **Dev**: Free tier (30GB) - easyescrow-dev
- **Staging**: Free tier or Fixed 250MB ($5/mo) - easyescrow-staging
- **Production**: Fixed 1GB ($12/mo) with HA - easyescrow-prod
