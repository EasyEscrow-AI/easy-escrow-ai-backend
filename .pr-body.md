## 🎨 Key Features

### SecretsManagementService API

```typescript
import { getSecretsManagementService } from './services';

// Initialize (called once at startup)
const secretsService = getSecretsManagementService();
await secretsService.initialize();

// Get a Solana keypair
const authorityKeypair = secretsService.getKeypair('AUTHORITY_KEYPAIR');

// Get an API secret
const jwtSecret = secretsService.getSecret('JWT_SECRET');

// Validate all required secrets
const validationResults = secretsService.validateSecrets();
```

### Supported Keypair Formats

1. **JSON Array** (Solana CLI default)
   ```json
   [1,2,3,4,5,...,64]
   ```

2. **Base58** (Solana standard)
   ```
   5J7XqR9pZt8F3nK4mH2wV9xQ1pY7cD8tA6bE9fG3hJ5kL2mN4pR6sT8vW1xY3zA5
   ```

3. **Base64** (Common encoding)
   ```
   AQIDBAUGBwgJCgsMDQ4PEBESExQVFhcYGRobHB0eHyAhIiMkJSYnKCkqKywtLi8...
   ```

## 🚀 Quick Start for Developers

### 1. Install Git Hooks
```bash
# Linux/Mac
bash scripts/setup-git-hooks.sh

# Windows
.\scripts\setup-git-hooks.ps1
```

### 2. Generate Required Secrets
```bash
# Generate Solana keypair
solana-keygen new --outfile authority-keypair.json --no-bip39-passphrase

# Copy the content
cat authority-keypair.json

# Generate other secrets
openssl rand -base64 48  # Receipt signing key
openssl rand -hex 32     # Webhook secret
```

### 3. Configure Environment
Create `.env.local` (automatically gitignored):
```bash
AUTHORITY_KEYPAIR='[1,2,3,...,64]'
RECEIPT_SIGNING_KEY='your-key-here'
DATABASE_URL='postgresql://localhost:5432/escrow_dev'
REDIS_URL='redis://localhost:6379'
```

### 4. Delete Keypair Files
```bash
# IMPORTANT: Delete after copying to environment
rm authority-keypair.json
```

### 5. Start Application
```bash
npm run dev

# Look for: "[SecretsManagementService] Secrets management initialized successfully"
```

- Delete keypair files after copying
