## Overview

The localnet environment provides a fast, free, and deterministic testing environment for the escrow program. It includes:

- **Local Solana Validator**: Running on `localhost:8899`
- **Test Keypairs**: Pre-generated accounts for buyer, seller, admin, and fee collector
- **USDC Token Mint**: Simulated USDC for testing escrow transactions
- **NFT Collection**: Test NFTs for escrow agreements
- **Test Matrix**: Comprehensive test scenarios including happy path, edge cases, and security tests

### Benefits of Localnet Testing

- ✅ **Fast**: No network latency or confirmation delays
- ✅ **Free**: No SOL required for testing
- ✅ **Deterministic**: Consistent results across test runs
- ✅ **Isolated**: Safe environment for testing edge cases and failures
- ✅ **Complete Control**: Fast epoch transitions and custom configuration

## Prerequisites

Before setting up localnet, ensure you have:

- ✅ Rust (installed via rustup)
- ✅ Solana CLI 2.1.13+ ([Installation Guide](SOLANA_SETUP.md))
- ✅ Anchor Framework 0.32.1+ ([Installation Guide](SOLANA_SETUP.md))
- ✅ Node.js 16+ and npm
- ✅ PowerShell (for Windows scripts)

Verify installations:

```powershell
solana --version      # Should show: solana-cli 2.1.13
anchor --version      # Should show: anchor-cli 0.32.1
node --version        # Should show: v16.0.0 or higher
```

## Quick Start

### 1. Start the Local Validator

In a new terminal window (keep it running):

```powershell
npm run localnet:start
```

Or directly:

```powershell
.\scripts\start-localnet-validator.ps1
```

This will start a local Solana validator on `http://localhost:8899`. Leave this terminal running for the duration of your testing session.

### 2. Setup the Localnet Environment

In a new terminal window:

```powershell
npm run localnet:setup
```

Or directly:

```powershell
.\scripts\setup-localnet.ps1
```

This script will:
- Generate test keypairs (buyer, seller, admin, fee-collector)
- Fund accounts with SOL
- Create and distribute test USDC tokens
- Save configuration to `.env.localnet`

### 3. Build and Deploy the Escrow Program

```powershell
# Build the program
anchor build

# Deploy to localnet
anchor deploy
```

Copy the program ID from the deployment output and update it in:
- `programs/escrow/src/lib.rs` - Update the `declare_id!` macro
- `Anchor.toml` - Update the program ID in `[programs.localnet]`

Then rebuild:

```powershell
anchor build
```

### 4. (Optional) Setup Test NFT Collection

```powershell
npm run localnet:setup-nfts
```

This creates 10 test NFTs owned by the seller account.

### 5. Run Tests

```powershell
# Run all localnet tests
npm run test:localnet

# Run comprehensive test suite
npm run test:localnet:comprehensive
```

## Detailed Setup

### Directory Structure

After setup, your project will have:

```
.localnet/                    # Test keypairs (gitignored)
  ├── buyer.json
  ├── seller.json
  ├── admin.json
  ├── fee-collector.json
  ├── usdc-mint.json
  ├── usdc-mint-authority.json
  ├── nfts/                   # NFT mint keypairs
  │   ├── nft-1-mint.json
  │   ├── nft-2-mint.json
  │   └── ...
  └── nft-collection.env      # NFT addresses

.env.localnet                 # Auto-generated config (gitignored)
test-ledger/                  # Validator ledger data (gitignored)
```

### Configuration Files

#### `.env.localnet`

Auto-generated by `setup-localnet.ps1`, contains:

```env
# Solana Configuration
SOLANA_NETWORK=localnet
SOLANA_RPC_URL=http://localhost:8899
SOLANA_WS_URL=ws://localhost:8900

# Test Keypairs
BUYER_KEYPAIR_PATH=.localnet/buyer.json
SELLER_KEYPAIR_PATH=.localnet/seller.json
ADMIN_KEYPAIR_PATH=.localnet/admin.json
FEE_COLLECTOR_KEYPAIR_PATH=.localnet/fee-collector.json

# Test Wallet Addresses
BUYER_ADDRESS=<buyer_public_key>
SELLER_ADDRESS=<seller_public_key>
ADMIN_ADDRESS=<admin_public_key>
FEE_COLLECTOR_ADDRESS=<fee_collector_public_key>

# USDC Mint Configuration
USDC_MINT_ADDRESS=<usdc_mint_public_key>
USDC_MINT_AUTHORITY_PATH=.localnet/usdc-mint-authority.json
USDC_MINT_DECIMALS=6

# Platform Configuration
PLATFORM_FEE_BPS=250
```

### Scripts Overview

#### `start-localnet-validator.ps1`

Starts the local Solana validator with optimized settings:

```powershell
solana-test-validator \
  --reset \              # Clean state on restart
  --quiet \              # Minimal logging
  --slots-per-epoch 32   # Fast epoch transitions
```

**Options:**
- Remove `--reset` to preserve state between restarts
- Adjust `--slots-per-epoch` for faster/slower epoch transitions

#### `setup-localnet.ps1`

Complete environment setup script that:

1. ✅ Creates `.localnet/` directory
2. ✅ Generates test keypairs
3. ✅ Configures Solana CLI for localhost
4. ✅ Checks validator status
5. ✅ Airdrops 10 SOL to each account
6. ✅ Creates USDC mint
7. ✅ Creates token accounts
8. ✅ Distributes 1000 USDC to buyer and seller
9. ✅ Saves configuration to `.env.localnet`

#### `setup-nft-collection.ps1`

Creates test NFT collection:

1. ✅ Mints 10 NFTs (SPL tokens with 0 decimals, supply of 1)
2. ✅ Assigns ownership to seller
3. ✅ Disables future minting (true NFT behavior)
4. ✅ Saves NFT addresses to `.localnet/nft-collection.env`

#### `reset-localnet.ps1`

Cleans up the localnet environment:

1. ⚠️ Removes all keypairs from `.localnet/`
2. ⚠️ Removes validator ledger data
3. ⚠️ Removes `.env.localnet`
4. ✅ Resets Solana CLI to devnet

**Warning:** This is destructive! You'll need to run setup again.

## Troubleshooting

### Validator Won't Start

**Problem:** `solana-test-validator` command not found

**Solution:**
```powershell
# Verify Solana installation
solana --version

# Add to PATH if needed
$env:PATH += ";$env:USERPROFILE\.local\share\solana\install\active_release\bin"

# Or reinstall Solana (see SOLANA_SETUP.md)
```

### Validator Already Running

**Problem:** "Port 8899 already in use"

**Solution:**
```powershell
# Find and kill the existing validator
Get-Process | Where-Object {$_.ProcessName -like "*solana*"} | Stop-Process

# Or restart with the existing validator
solana cluster-version  # Verify it's running
```

### Setup Script Fails

**Problem:** "Local validator is not running"

**Solution:**
```powershell
# Make sure validator is running first
npm run localnet:start
# Then in another terminal:
npm run localnet:setup
```

### Test Failures

**Problem:** "Program not deployed" errors in tests

**Solution:**
```powershell
# Build and deploy the program
anchor build
anchor deploy

# Make sure you updated the program ID in:
# - programs/escrow/src/lib.rs (declare_id!)
# - Anchor.toml ([programs.localnet])

# Then rebuild
anchor build
```

### Connection Refused

**Problem:** "Connection refused to localhost:8899"

**Solution:**
```powershell
# Check if validator is running
solana cluster-version

# Check Solana config
solana config get
# Should show: RPC URL: http://localhost:8899

# Set if needed
solana config set --url localhost
```

### Insufficient Funds

**Problem:** "insufficient lamports" errors

**Solution:**
```powershell
# Airdrop more SOL to test accounts
solana airdrop 10 --keypair .localnet/buyer.json
solana airdrop 10 --keypair .localnet/seller.json
```

### Stale State

**Problem:** Tests fail due to previous state

**Solution:**
```powershell
# Stop validator (Ctrl+C)
# Restart with --reset flag (default behavior)
npm run localnet:start

# Or manually
solana-test-validator --reset
```

## Advanced Usage

### Custom Validator Configuration

Edit `scripts/start-localnet-validator.ps1`:

```powershell
solana-test-validator `
    --reset `
    --quiet `
    --slots-per-epoch 32 `
    --limit-ledger-size 50000000 `     # Limit ledger size
    --rpc-port 8899 `                  # Custom RPC port
    --faucet-port 9900 `               # Custom faucet port
    --ledger test-ledger `             # Custom ledger path
    --bpf-program <PROGRAM_ID> <SO_PATH>  # Preload programs
```

### Preserving State Between Restarts

Remove the `--reset` flag to preserve ledger state:

```powershell
# In start-localnet-validator.ps1
solana-test-validator `
    --quiet `
    --slots-per-epoch 32
```
