## Naming Convention

### Why DEVNET_STAGING_* ?

**Problem**: Without distinct naming, DEV and STAGING environments conflict:
- ❌ Both use `DEVNET_PROGRAM_ID`
- ❌ Both use `DEVNET_SENDER_PRIVATE_KEY`
- ❌ Cannot run both environments simultaneously
- ❌ Risk of using wrong configuration

**Solution**: Use `DEVNET_STAGING_*` prefix for STAGING:
- ✅ `DEVNET_STAGING_PROGRAM_ID` (STAGING only)
- ✅ `DEVNET_STAGING_SENDER_PRIVATE_KEY` (STAGING only)
- ✅ Clear distinction in logs and errors
- ✅ Can run DEV and STAGING side by side
- ✅ Prevents accidental misuse

### Variable Naming Patterns

| Environment | Program ID Variable | Sender Key Variable |
|-------------|--------------------|--------------------|
| **DEV** | `DEVNET_PROGRAM_ID` | `DEVNET_SENDER_PRIVATE_KEY` |
| **STAGING** | `DEVNET_STAGING_PROGRAM_ID` | `DEVNET_STAGING_SENDER_PRIVATE_KEY` |
| **PROD** | `MAINNET_PROGRAM_ID` | `MAINNET_SENDER_PRIVATE_KEY` |

## Security Implementation

### Secret Classification

**🔴 SECRET Variables** (25 total):
- Encrypted in DigitalOcean App Platform
- Never committed to git
- Rotated quarterly
- Backed up securely

**🟢 PUBLIC Variables** (45 total):
- Can be stored in plain text
- Configuration values
- No sensitive information

### Secret Rotation Schedule

| Secret Type | Frequency | Impact |
|-------------|-----------|--------|
| Wallet Private Keys | Quarterly | High |
| Database Passwords | Quarterly | Medium |
| Redis Passwords | Quarterly | Medium |
| JWT Secrets | Quarterly | Medium |
| Webhook Secrets | Quarterly | Low |
| API Keys | Quarterly | Low |

### Security Best Practices Implemented

✅ **Never commit secrets to git**
- `.env.staging` in `.gitignore`
- Only example files committed

✅ **Use encrypted storage**
- DigitalOcean App Platform secrets
- Marked as `type: SECRET` in app spec

✅ **Automated rotation**
- Script generates new secrets
- Timestamped backups
- Verification steps

✅ **Principle of least privilege**
- Database user has minimal permissions
- Separate credentials per environment

✅ **Audit trail**
- Rotation summaries saved
- Backup of old secrets
- Timestamped changes
